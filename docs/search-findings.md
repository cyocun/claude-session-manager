## 失敗パターン分類

0 件ヒット or `escape_no_open` のクエリを観察して分類する。

### A. キーワード一致で拾えなかった（= 意味検索で救えそう）

- 例: 「Tauri のビルドで詰まった件」→ ヒットなし（実際のセッションでは「WebKitGTK 入ってない」と議論していて「ビルド」「詰まる」は未登場）

## 意味検索で救える / 救えないケースの見積もり

A カテゴリーの件数を元に:

- **救える見込み**: A の割合 × 月次検索回数 × ユーザー数 = 月間 X 件の成功追加見込み
- **救えないもの**: B / C は意味検索ではなくフィルタ / インデックス拡張の問題

## 実装コスト見積もり

Phase 5 候補:

1. **sqlite-vec + fastembed-rs** — ローカル完結、初回インデックス重い
   - バイナリサイズ増: +15-40 MB（埋め込みモデル）
   - 初期インデックス時間: N msg × ~20ms = 100ms/session 想定
   - 増分更新: update_search_index 相当のフックで対応可能
2. **API embedding（OpenAI / Voyage / Cohere）** — ネット必須、月額コスト
   - コスト: $0.02/1M tokens × 10k msg × 200 tokens = $0.04/全セッション
   - 秘匿性: セッション内容を外部に送ることになる → ユーザーの許容次第
